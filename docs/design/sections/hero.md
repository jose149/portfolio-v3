# Hero section design

## Purpose

The Hero introduces the portfolio with confidence, elegance and clarity.

It should communicate:

- frontend engineering competence
- product thinking
- premium visual taste
- trustworthiness
- attention to detail

## Layout

Desktop:

- left column: text and CTAs
- right column: portrait or architectural visual
- maximum content width around 1280px
- generous vertical spacing
- strong editorial hierarchy

Mobile:

- one column
- text first
- visual second
- CTAs stacked or wrapped naturally

## Content structure

- Eyebrow: `Frontend Engineer · Product-minded Developer`
- Main headline: `I design. I build.`
- Highlight: `I ship results.`
- Value proposition:
  `I build elegant, accessible and high-performance digital products that turn complex requirements into clear user experiences.`
- Primary CTA: `Let’s connect`
- Secondary CTA: `View projects`
- Metadata: `Mallorca · CET/CEST · Remote-friendly`

## Visual direction

Use the approved portrait if available.

Preferred asset:

`/images/portrait-hero.png`

The visual should feel:

- professional
- editorial
- minimal
- warm-neutral
- premium
- slightly artistic

Avoid:

- stock developer illustrations
- laptops with code
- excessive gradients
- loud decorative effects
- generic CV layout

## Theme

Use only design-system tokens.

Do not introduce new colour constants unless approved.

## Motion

Allowed:

- subtle CTA hover
- subtle visual reveal
- line or mask reveal if already supported

Not allowed:

- large parallax
- scroll hijacking
- constant looping animation
